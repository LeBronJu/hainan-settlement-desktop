# 阶段二模板候选浏览器研究（2026-07-23）

状态：当前模块 note。本文记录新增主体面对大量可借用分表模板时已经落地的候选浏览和搜索合同。

## 当前行为

- 海南或重庆新增主体没有同名精确模板、但存在多份可读同费用类型模板时，会把全部候选加入一个必选预检项目，由操作员显式选择。
- 候选全集进入预检签名，最终模板路径必须属于本次预检候选；模板目录和 workbook 内容还受输入指纹保护。
- WPF 把完整候选集与当前可见批次分开：进入预检卡时洗牌一次，每批最多显示 5 份，同一轮内不重复；可查看上一批/换一批，轮末重新打乱。
- `搜索全部模板` 在本次完整冻结候选集上按主体、负责人和文件名过滤，选中后回到主体卡；不允许借此加入目录外文件。
- 已选模板不随换批或重新打乱清除，仍按完整候选集校验。
- 广东当前只有独立分表月份初始化，没有完整阶段二模板选择流程。

## 当前交互

采用“5 个一组的随机推荐 + 搜索全部模板”的组合方式，不自动选择任何模板。

主体卡内显示：

- 候选总数和本轮进度，例如“已查看 5 / 327，剩余 322”。
- 最多 5 个候选，按主体、负责人、文件名分栏或分行显示，完整路径继续放在折叠的技术详情中。
- `上一批`、`换一批`（轮末显示 `重新打乱`）和 `搜索全部模板`。
- 已选模板固定显示；浏览其它批次时不清除选择，选择另一个候选时才替换。

随机分批不是每次独立抽样，而是进入预检卡时先把候选全集洗牌一次，再按 5 个连续分页。这样可以保证一轮内不重复：

| 候选数 | 本轮显示 |
| --- | --- |
| 1 | 维持现有唯一模板自动借用和复核提示，不进入人工选择 |
| 2—5 | 一次显示全部，不显示“换一批” |
| 7 | `5 + 2` |
| 13 | `5 + 5 + 3` |
| 全部看完 | 显示“重新打乱”；新一轮允许再次出现旧候选，但新一轮内部仍不重复 |

零候选、多份同名精确历史模板、模板不可读等现有阻断规则不变，不能通过随机抽样绕过。

## Excel 式搜索

`搜索全部模板` 打开一个独立候选选择窗口，行为接近 Excel 筛选：

- 顶部固定搜索框，下面是可滚动、虚拟化的结果列表。
- 列显示 `主体`、`负责人`、`文件名`；费用类型已由当前预检卡限定，不需要重复筛选。
- 输入一个关键词时，在主体、负责人和文件名三个字段中做不区分大小写的包含匹配。
- 输入多个由空格分隔的关键词时采用 AND 语义：每个关键词都必须在任一可见字段中命中。例如 `薛万甲 鄢桂兰` 可以匹配“主体包含薛万甲、负责人包含鄢桂兰”的候选。
- 中文、数字和英文文件名均按原文本匹配；忽略关键词首尾及连续空格。
- 实时显示“找到 N 个”，无结果时给出清空搜索提示。
- 支持方向键移动、Enter 确认、双击选择和 Esc 关闭；选择后回到主体卡并显示已选模板。
- 搜索范围始终是本次已预检并签名的候选全集，不把用户输入当作任意文件路径。

几百条内存候选对 WPF 过滤和虚拟化不是性能瓶颈。可以对输入变化做短延迟合并，避免每次按键立即重复刷新，但不需要数据库或额外索引服务。

## 安全和完整性边界

- Core/Excel 继续生成、校验并签名完整候选集合；随机分批和搜索只属于 WPF 展示状态。
- ViewModel 必须区分“完整候选集合”和“当前可见批次”。最终选择应对完整集合校验，不能因为换批后候选暂时不可见而失效。
- 随机顺序不需要进入预检签名；完整候选集合和最终选中的路径需要继续进入现有授权链。
- 用户在搜索窗口选择后，仍生成同一个 `Stage2TemplateDecision`，Excel writer 不需要知道候选是从随机批次还是搜索结果选出的。
- 如果未来允许从模板目录之外浏览任意 `.xlsx`，不能在原预检上直接继续。必须先校验费用类型、目标/来源月份和 workbook 结构，把文件加入新候选集合，并重新生成预检签名和输入指纹。

## 跨省实现边界

候选浏览器适合做成共享 WPF 展示组件，但省份规则不得进入公共 UI：

- 各省 Excel adapter 决定候选如何发现、何时允许借用、哪些情况阻断。
- 共享层只接收已经校验的候选，负责分页、随机顺序、关键词过滤和返回选择。
- 海南和重庆都已启用；广东完整阶段二未来接入时仍须由广东 Adapter 单独决定，不能因共享 UI 存在就自动开启。
- 候选合同后续宜从单纯路径升级为结构化选项，至少包含稳定路径、主体、负责人、文件名和费用类型，避免公共 UI 继续解析海南专属目录命名。

随机分批解决的是可读性，不会减少预检扫描、打开 workbook 或输入指纹哈希的成本。广东完整阶段二如果面对几百份模板，扫描性能应作为独立任务测量和优化，不能通过只校验随机抽中的 5 份来换取速度。
