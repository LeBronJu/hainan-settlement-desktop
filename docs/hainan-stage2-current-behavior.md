# 海南阶段二当前行为

状态：当前行为文档。

本文件记录 C# 桌面版海南阶段二当前可依赖行为。它收敛了 `CONTEXT.md`、`HANDOFF.md`、阶段二测试、`HainanStage2SettlementGenerator` 及其内部 Excel 组件，和 2026-06 到 2026-07 授权真实工作副本对照得到的结论。

不要把本文件理解为重庆阶段二规格。重庆可以借鉴工程形态，但不能直接套用海南列位、单位、公式、支付方默认或目录结构。

## 输入和输出

输入：

- 人工整理后的当月海南台账。
- 上月代理分表文件夹。
- 上月居间分表文件夹。
- 上月或人工修正版汇总表。
- 结算月份和输出目录。

输出：

- `2026年代理 - 海南` 下的代理分表。
- `2026年居间 - 海南` 下的居间分表。
- `【2026年海南省代理费汇总表-X月自动化】.xlsx`。
- `X月结算生成总报告.json`。
- `阶段二校验报告.txt`。
- `自动生成汇总提示.txt`，每批固定输出；没有自动新增或需复核提示时明确写“本批无相关提示”，避免同月重跑残留旧报告。

程序不生成 `X月阶段二结算路由.json`，审计 JSON 也不作为下月付款路由输入。以上文件先写到本批唯一 staging；全部核验通过后才一起发布到上述正式位置。中途失败的内容保留在醒目的 `【未完成-禁止付款】...` 目录，不能作为正式结算结果。

## 台账读取

- 阶段二以人工整理后的台账为准。
- 不自动补负责人、项目开发人、代理/居间主体或付款信息。
- 不为了匹配汇总表或收款账户名称而修改台账客户名称。
- 海南台账电量单位是 `万千瓦时`。
- 代理/居间金额读取按海南台账月度区块和现有列位实现；重庆不能复用该列位。
- 代理/居间关系在按金额过滤前读取原始主体、比例、单价和税率状态：主体和三项参数全空表示无关系；主体已填时三项必须是大于 0 的数字；主体空但参数有内容、主体已填但参数缺失/非数字/小于等于 0 都会阻止生成。零电量、零金额行也参与校验。
- 汇总身份键为 `(标准化主体, 代理费/居间费)`。同一身份跨负责人出现时合并为一组，唯一负责人取该关系在台账第一次出现的物理行；第一行负责人为空会阻止生成，多负责人则提示复核。程序不读取或自动设置台账/汇总表橙色。
- 同一身份所有有效关系行（包括零金额行）必须只有一个扣税率；多个不同税率会阻止生成。

## 分表生成

- 分表生成是模板驱动。精确模板按 `(费用类型, 主体)` 查找，不包含负责人；唯一精确模板可以跨负责人目录复用，多个精确候选会阻止生成。
- 没有精确模板时，只有唯一可靠同类型模板可以借用并提示复核；无候选或多个候选都不会静默取第一个。
- 存量主体保留历史月份 sheet。
- 新增代理/居间主体如果借用同类型模板，输出副本只保留本次生成的目标月 sheet，不携带模板来源人的历史月份 sheet。
- 存量主体新增客户会写入同一主体分表并进入预检复核，不拆出新汇总行；模板中有、本月台账没有的特殊明细进入复核且不自动继承到本月 sheet。
- 目标月 sheet 写入客户名称、电量、比例、单价、税率等本月输入值，并重写合计行公式范围。
- 若合计行缺少边框、对齐等表格格式，优先从前序月份合计行修复，取不到时用最后一条明细行兜底。
- 分表底部已有签字日期时顺延一个月；日期可以是文本，也可以是 Excel 日期值。
- 模板没有底部签字日期时不新增日期。
- 上月分表存在人工退补、补扣或历史修正性质的特殊明细行，而本月台账同主体下找不到对应客户/描述时，预检提示；生成本月分表时不继承该行。

## 汇总表生成

- 汇总表模板按 sheet 名或标记识别主汇总表、清能汇总表、清辉汇总表。
- 海南汇总表右侧长期字段以 `累计代理费总计` 作为定位锚点。
- 生成目标月时，没有目标月区块就在 `累计代理费总计` 前插入 6 列；唯一目标月区块已存在时保留结构并安全重写；重复目标月区块会阻止生成。
- 新目标月区块复制前一个月区块格式，再把前一个月区块隐藏，目标月区块取消隐藏。
- 目标月区块标签为：`代理费`、`居间费`、`退补电费`、`当月抵扣`、`费用合计`、`当月实际支付`。
- 汇总表只把第 4 行到 `合计` 行之前视为主体数据。`合计` 行之后的签字、审核或页脚内容不得当成主体。
- 汇总表签字日期默认写为结算月份加 2 个月的 8 日。
- 主汇总表是长期业务字段的 canonical 来源。清能/清辉分 sheet 按支付方保留且互斥；所属 sheet 的业务行从主表同步，不属于该支付方的主体行删除，不能继续保留与主表分叉的收款人、委托、发票、借支/抵扣或备注。
- 主表或支付方 sheet 出现重复汇总键、支付方 sheet 有而主表没有的孤儿键、同一键同时属于两方、固定支付方 sheet 缺失等无法可靠同步的结构问题会阻止生成。

## 汇总主体继承和默认值

存量汇总主体：

- 以主汇总表整行继承长期资料：D/E/G 等固定资料，以及右侧借支、已抵扣、剩余未抵扣、抵扣说明、还完月份、支付方、备注等状态。
- F 列收款人按完整单元格文本继承；不拆分多人姓名。比较冲突只忽略首尾空白/换行和换行编码，写入仍保留选定来源原文。
- H 列重写为 `J-I` 公式，I 列写台账唯一扣税率，J 列固定 13%，K 列写首次关系行负责人；目标月金额、抵扣和必要公式按本月重写。
- 主表与可靠支付方来源的不同非空收款人文本会阻止生成；空白加唯一非空值采用该非空原文并提示，所有来源都空时保持空白并提示人工完善。
- 支付方从可靠历史字段/所属关系继承；无法可靠继承的空白或不支持值必须在预检选择清能/清辉，冲突值会阻止生成，不再默认清辉。

新增汇总主体：

- 复制模板行格式后清空内容。
- 自动写入：
  - `是否委托收款=否`
  - `收款人=主体名称`
  - `发票票种=平台`
  - `税率=合计税率-扣税率` 公式
  - `扣税率=台账唯一税率`
  - `合计税率=0.13`
  - `负责人=首次关系行负责人`
  - `代理/居间新增=目标月`
- 支付方不再静默默认 `清辉`。除下方海南专属硬编码例外外，新增汇总主体会在预检窗口要求本次显式选择 `清能` 或 `清辉`；如果调用方绕过预检或没有传入选择，生成器会拒绝生成。
- 当前代码有一个海南专属硬编码例外：`海南精研科技有限公司` 的 `代理费` 从 3 月起支付方为 `清能`。
- 该专属规则优先于其更早的清辉历史状态；历史清辉记录不会被通用支付方冲突误拦截，生成时统一同步到清能。
- 自动新增主体会写入提示，包含本次选择的支付方，并要求人工复核。

## 抵扣和金额

- 目标月代理费/居间费金额来自分表自算结果。
- `退补电费` 列目前由海南阶段二汇总生成清空，不作为海南代理/居间自动计算输出。
- 当月抵扣按借支、已抵扣、剩余未抵扣和抵扣说明中的“每月扣除”规则计算。
- `费用合计` 和 `当月实际支付` 写公式。
- 台账与分表金额核对容差为 `0.0001` 万元；该量级差异通常按四舍五入差异处理，仍写入校验报告供人工看。
- 当前不接入桌面 Excel 自动化强制重算，只使用 ClosedXML 写公式和缓存。

## 预检和校验

生成前预检提示：

- `阻断`：关系主体/参数不完整或非正数、首次负责人缺失、同一身份多个税率、重复/不可读/歧义模板、汇总键/收款人/支付方来源冲突、汇总结构不能可靠识别。存在阻断时确认按钮禁用。
- `必须选择`：新增汇总主体或存量主体支付方无法可靠继承时，在对应项目行内选择 `清能` / `清辉`；未全部选择时确认按钮禁用。
- `需复核`：新增客户/主体、多负责人、唯一模板借用、参数合法变化、上月特殊明细不继承、台账与分表自算金额差异、可靠来源收款人缺失等。程序可完整生成，但完成后继续提醒检查。
- `信息`：不影响生成的说明。
- 台账与分表金额比较在生成前完成，不能等写完文件才首次提示。金额差异确认后，分表和汇总仍统一采用程序按关系参数自算的结果；自算值恰为 0 时也不回退台账金额。
- 海南和重庆共用 WPF 预检窗口和四种处置语义，海南 Excel Adapter 仍独立解释台账列、模板、公式、sheet 和精研例外。
- `Analyze` 只读且不授权生成；workflow 初次 plan 冻结预检签名和文件状态。台账、模板目录、汇总模板、月份/输出配置，以及本批规划旧分表或旧汇总表在确认后变化时，正式生成会拒绝并要求重新预检。直接绕过 plan/确认调用生成器同样会被拒绝。

生成后校验：

- 写 `阶段二校验报告.txt`。
- 报告包含预检变化、负责人缺失提示、模板提示和台账/分表金额核对差异。
- 如果确认台账金额才是正确结果，需人工同步检查或修改分表和汇总表。

## 正式输出完整性

- 预检通过后才创建 staging，所有分表、汇总表和报告先写到隔离区。
- 代理/居间分表是年度历史 workbook，程序不自动删除不活跃主体整本文件。生成前会扫描受管输出根：非本批计划路径的 workbook 只含旧月份时保留；若仍含本次 `X月` sheet 或无法读取，则先阻断，避免同月重跑把旧主体混入本批。
- 本批规划路径已经存在且含目标月 sheet 时作为同月重跑复核项，允许事务式覆盖；但该旧 workbook 或正式汇总表在预检确认后又被修改时，本次拒绝覆盖并要求重新预检。
- 发布前按预期 `(主体, 费用类型)` 集合核验：每组恰有一份可重开的分表，目标月份/主体/明细和合计金额正确；主汇总表恰有一行，H/I/J、负责人、金额正确；清能/清辉所属互斥且对应行与主表一致；审计报告存在且其中路径已经映射为最终正式路径。
- 任一核验失败都不发布半批正式文件。失败 staging 改名并写 `【未完成-禁止付款】说明.txt`；正式目录已有同名文件时，发布过程先备份，失败会尽力回滚。
- 当前没有“生成其余正确主体”或“部分汇总表”按钮。失败目录可用于排查，不能付款；广东独立 workbook 的部分完成策略不适用于海南正式结算。
